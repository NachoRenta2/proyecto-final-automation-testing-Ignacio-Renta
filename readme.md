# Proyecto de Talento Tech<br><br>## Proposito del proyecto<br>Este proyecto tiene como objetivo automatizar pruebas de UI y de API para el sitio **SauceDemo**, aplicando practicas como Page Object Model, manejo de datos externos, generacion de reportes HTML, logging y captura automatica de pantalla.<br><br>## Tecnologias utilizadas<br>- Python 3.x<br>- Pytest<br>- Selenium WebDriver<br>- Logging<br>- Faker<br>- CSV / JSON<br>- Request<br><br>## Reportes y Logs<br><br>El proyecto genera tres tipos principales de resultados durante la ejecucion de las prubas: **reporte HTML**, **capturas de pantalla**, **archivo de log**<br><br>### Reporte HTML<br>Se genera un reporte HTML detallado con el nombre de ```reporte.hmtl``` en la **carpeta raiz** del proyecto<br><br>### Logs de ejecuciÃ³n<br>Tambien se genera un log con informacion detallada de toda la ejecuciÃ³n de las pruebas en la siguiente ubicacion: ```logs/suite.log```<br><br>### Capturas de pantalla<br><br>Se realizan capturas de pantalla por cada test que haya fallado y se encuentran en la siguiente ubicacion: ```reports/screens/```<br><br>## Ejuctar todas las pruebas<br>Para iniciar la ejecucion de las pruebas debes ejecutar la siguiente linea:<br><br>```bash<br>python -m run_test.py<br>```<br><br>## Â¿Como interpretar los reportes?<br>- Al ejecutar `run_test.py`, se genera un archivo HTML en la carpeta raiz.<br>- El reporte incluye:<br>    - Lista completa de test ejecutados<br>    - El estado de cada prueba<br>    - La duracion de cada test<br>    - Las capturas de pantalla para pruebas fallidas<br><br>## Pruebas incluidas<br>- Login exitoso y fallido<br>- Login exitoso y fallido usando faker<br>- Comportamiento de la pagina de inventario<br>- Comportamiento de la pagina del carrito<br>- API (Reqres): GET users, POST create user, DELETE user, validaciones de codigos HTTP, validaciones de estructura JSON<br><br>## Manejo de datos de prueba<br>- En la carpeta `datos` se incluyen archivos como:<br>    - `data_login.csv` -> datos de usuarios validos o invalidos<br>    - `productos.json` -> datos de productos para validacion<br><br>### Conclusion<br>Este proyecto ofrece una estructura organizada y escalable para automatizar pruebas de API utilizando Python y Pytest. Incluye un flujo simple de ejeucion mediante `run_test.py`, generacion automatica de reporte HTML facilitando el analisis de las pruebas.<br><br>La arquitectura del proyecto esta pensada para agregar nuevos casos de prueba y configuraciones sin modificar el nucleo del proyecto, manteniendo buenas practicas y permitiendo su escalabilidad en el tiempo.
